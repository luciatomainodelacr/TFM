11.A. if/elif/else¶

Python implementa la lgica if/else de forma muy sencilla. Simplemente, comprueba si la condicin del if es True. Si no lo es, se ejecuta el else. La sintaxis es:

Ni siquiera es obligatorio incluir un else con el if:

Como no se cumple el if y no hay un else, no ocurre nada.

Pero Python no inlcuye la funcionalidad que encontramos en otros lenguajes llamada switch o match. Por el contrario, esta lgica se implementa con la palabra elif. Hace que el resultado sea muy sencillo de leer por el ser humano:

elif otra_condicion_distinta:
    # Hacer otro algo
elif otra_otra_condicion_distinta:
    # Hacer otro otro algo
else:
    # Hacer otro otro otro algo

Podemos incluir tantos elif como queramos entre el if y el else. Para ejecutar el cdigo, Python sigue el procedimiento de arriba abajo:

  • Comprueba si el if es True. Si lo es, lo ejecuta y se olvida del resto de abajo (no comprueba ni los elifs ni el else).
  • Si el if es False, pasa a comprobar el primer elif.
  • Si este primer elif es True, lo ejecuta y se olvida del resto de abajo (tampoco comprueba el resto de elifs ni el else).
  • Si no lo es, pasa al siguiente elif.
  • ... Y as. Si ningn elif es True, se terminar por ejecutar el else.

Veamos un ejemplo:

b = 3

    print("A!")
elif a < 3 and b > 4:
    print("B!")

    print("C!")
else:
    print("Ninguna de las anteriores")

Una caracterstica determinante de los diccionarios y sets (en comparacin con el resto de colecciones/iterables, como pueden ser las listas, tuplas o strings) es el tiempo de acceso a un determinado elemento. Para mostrar esto, supongamos que queremos ver si un determinado elemento est dentro de una lista:

Vemos que es ms rpido; y las diferencias se acentan an ms a medida que creciera el tamao de la coleccin. Por qu? Resulta que cuando le pedimos a Python que compruebe si un elemento existe en una lista, va recorriendo la lista, elemento por elemento, de izquierda a derecha, de forma que el tiempo que tarda en comprobar si el elemento est es $O(n)$, donde $n$ es la len() de la lista.

Funciona. Pero cmo? Pues bien: en el fondo, reduce va iterando por lista_a_sumar, y aplicando sumar_reduce:

  • Coge el primer nmero de la lista (el 1), y lo pasa como el argumento siguiente a sumar_reduce. resultado_acumulado es 0, ya que no hemos definido un "valor inicial del resultado", acorde al punto 3 que hemos puesto antes.   • En la seguna iteracin, coge el segundo nmero de la lista (el 2), y vuelve a aplicar sumar_reduce. Ahora, resultado_acumulado es 1 (el resultado de la iteracin anterior), y siguiente es 2. Los suma 1 + 2 = 3. Y pasa a la siguiente iteracin.
  • Y as hasta que no quedan ms elementos en la lista. Efectivamente, el resultado final es la suma de todos los nmeros de la lista.
