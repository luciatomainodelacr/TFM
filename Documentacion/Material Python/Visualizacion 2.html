1. Gráficos web sin D3: Bokeh¶

Bokeh es otra biblioteca de visualizacin basada en web que nos permite hacer grficos bonitos e interactivos con poco esfuerzo, a partir de arrays de Numpy o Series y DataFrames de pandas.

A diferencia de D3, Bokeh no utiliza elementos SVG para dibujar en el navegador: sino que los genera en <canvas> de HTML5, lo cual los hace mucho ms ligeros, y ms apropiados si tenemos que representar una gran cantidad de datos. Por otro lado, el grfico se genera ms de la parte del servidor que la del cliente.

Bokeh se puede utilizar tanto en Notebooks de Jupyter, como en un servidor dedicado que sirva las visualizaciones a cualquier pgina web. Aqu nos vamos a centar en utilizarlo con los Notebooks.

Para instalar Bokeh, como siempre, podemos utilizar pip:

$ pip install bokeh

Una vez hecho: la primera vez que queremos que aparezca Bokeh en el Notebook, debemos inicializarlo:

Ahora bien, utilizar Bokeh es algo distinto a utilizar Matplotlib o Seaborn, pero tampoco demasiado. Al igual que en estos dos, debemos empezar por definir la figure en Bokeh (que en Matplotlib y Seaborn es opcional y lo har por nosotros si no lo hacemos, pero en Bokeh solo en algunos casos):

In [3]:
from bokeh.plotting import figure as bokeh_figure, show # Le ponemos un alias a la
                                                        # importacin de figure de
                                                        # Bokeh, para no confundir
                                                        # con el elemento figure
                                                        # de Matplotlib.

# Generamos un par de arrays de Numpy:
datos_x = np.arange(0, 50)
datos_y = np.random.randint(0, 100, 50)

# Hacemos una nueva figure de bokeh:
figura_bokeh = bokeh_figure(title="Ejemplo de Bokeh", x_axis_label='Eje x', y_axis_label='Eje y')

# Generamos una lnea con los datos generados,
# y le damos un ancho de 3:
figura_bokeh.line(x=datos_x, y=datos_y, legend=u"Lnea 1", line_width=3)

# Mostramos la figure con show:
show(figura_bokeh)
pass

Al igual que mpld3, Bokeh nos ofrece pequeos widgets para hacer zoom y movernos por el grfico, y nos permite cambiar el tamao del grfico con el de "resize" una vez generado.

A pesar de que tiene poco ms de ao y medio, Bokeh es ya una biblioteca de visualizacin MUY extensa y de relativamente bajo nivel (quizs no tanto como Matplotlib, pero cada da se acerca ms). Adems, evoluciona increblemente deprisa; principalmente porque Continuum Analytics (la empresa que distribuye Anaconda) est siendo su principal promotora.

Si bien se podra dedicar casi un mdulo entero a Bokeh y vamos a dejar muchas de sus funcionalidades como trabajo "de campo" para el alumno, vamos a explicar a alto nivel los elementos principales de Bokeh.

Aparte del propio elemento figure (que al igual que en Matplotlib es como "el marco" de nuestros grficos), Bokeh define principalmente tres elementos:

  • Charts: los elementos y visualizaciones estadsticos de alto nivel: barplots, grficos de tarta, boxplots, mapas de calor, histogramas...
  • Glyphs: son elementos ms a bajo nivel: crculos, lneas como la del primer ejemplo mostrado y similares. Dentro de la API se encuentran como mtodos sobre la clase Figure, parte de bokeh.plotting: de forma que como en el primer ejemplo, primero definimos la figura, y luego llamamos a mtodos de ella (como figura_bokeh.line()) para generarlos. Personalmente me parece una forma terrible de estructurar la biblioteca, pero para gustos los colores...
  • Widgets y otras interacciones: Bokeh a da de hoy es ms una aplicacin web que nicamente una biblioteca de visualizacin. Su extensa API construida sobre su propia biblioteca de Javascript permite, desde Python, generar una compleja lgica de interaccin con los grficos a travs de la creacin de widgets (como lupas de zoom, sliders, botones...), linkeado de varios grficos (para que cambien a la vez cuando interactuamos con uno de ellos), incluso la generacin de callbacks de Javascript (por ejemplo con AJAX) para dotar a tus visualizaciones de prcticamente todo lo que ofrece HTML5 en los navegadores modernos.

from bokeh.charts import Bar

# http://bokeh.pydata.org/en/latest/docs/reference/palettes.html#bokeh-palettes
from bokeh.palettes import BrBG9

# En el fondo, los Charts de Bokeh hacen operaciones "por debajo",
# como son los groupby y value_counts:
barplot = Bar(data=df_padre[df_padre["tiempo trabajando"] >= 0], # Filtramos valores con pandas
              values="tiempo trabajando", # Nos centramos en la columna tiempo trabajando
              label="tiempo trabajando", # Hacemos un groupby por tiempo trabajando tambin (eje x)
              agg="count", # Queremos la cuenta
              stack="padre", # Queremos que en cada barra, nos stackee padre/no padre
              legend="top_right",
              color=BrBG9, # La palera de color importada
              title=u"Tiempo trabajado (frecuencia): Padre/No Padre",
              xlabel="Tiempo trabajando",
              ylabel="Cuenta",

2.A. Widgets en Jupyter¶

Resulta que gracias al poder de Jupyter, tambin podemos hacer widgets de slider que interacten con Matplotlib (por ejemplo). Gracias a la biblioteca ipywidgets podemos generar este tipo de interacciones. Podemos descargarla, como siempre, con pip:

$ pip install ipywidgets

Aqu est el Github del proyecto.

La mayora de lo que podemos hacer lo obtenemos de ipywidgets.interactive:

In [11]:
from ipywidgets import interactive

nicamente tenemos que incluir el contenido interactivo en una funcin, que tome como argumentos aquellos elementos que van a ser interactivos. Una vez definida nuestra funcin no tenemos ms que llamar a interactive(nuestra_funcion, los_argumentos_que_puede_aceptar).

Podemos ver que en funcin de cmo pasemos los posibles argumentos (es decir, las posibilidades de cada variable que genera un widget) cada widget se adapta en forma para ofrecer la mejor interaccin posible (slider, checkbox, seleccin)...

Nota: es posible que los widgets no aparezcan en la versin exportada (HTML) del notebook; y an as, no sern funcionales. Para poder utilizar los widgets, es necesario abrir el notebook (el archivo .ipynb) y ejecutarlo.

def dibujar_histograma(numero_de_bins=5, color="blue", horizontal=False):
    # Configuramos el valor orientacion que le vamos a 
    # pasar luego a pandas para el plot:
    orientacion = "vertical"
    if horizontal == True:
        orientacion = "horizontal"

    # Y dibujamos en pandas (dentro de la misma
    # funcion):
    df_padre["edad"].plot(kind="hist",
                          bins=numero_de_bins,
                          color=color,
                          figsize=(8,6),
                          orientation=orientacion  # Los histogramas de pandas
                                                   # admiten el valor orientation
                                                   # que puede ser "horizontal"
                                                   # o "vertical" (por defecto). 
                        )

interactive(dibujar_histograma, numero_de_bins=(1, 100), color=["blue","red","green","yellow"], horizontal=False)

def generar_nan(celda):
    """Va por cada celda de un dataframe,
    con la posibilidad del 30% de que dicha
    celda se convierta en NaN"""
    numero_random = np.random.randint(0,10)
    if numero_random <= 3:
        return np.nan
    else:
        return celda

# La aplicamos con applymap():

2.D. Otras bibliotecas de visualización¶

Cada vez aparecen ms y ms bibliotecas de visualizacin en Python. La mayora de las nuevas estn parcial o totalmente orientadas a la visualizacin web. Basta con buscar en Internet para encontrar, literalmente, ms opciones de las que un ser humano equilibrado puede llegar a utilizar a lo largo de su vida.

Dicho esto, recomiendo echar un vistazo a las siguientes tres:

  • Lightning: De momento es muy joven, pero apunta a ser de las mejores para visualizaciones en tiempo real de forma sencilla; y compatibles con muchos lenguajes.
  • Pygal: Escasa en funcionalidad, pero es probablemente la ms bonita de cuantas he visto. Incluye grficos muy curiosos.
  • Plotly: Es bonita y sencilla, y bastante famosa. Ofrece herramientas para gente que no sabe programar; pero tambin ofrece una API de Python para los que s saben (como t!). Puedes comenzar leyendo sobre Plotly aqu
  • Bqplot: Desarrollada por Bloomberg, es muy reciente (sali hace tres meses aproximadamente). Permite generar visualizaciones interactivas en los Notebooks, al igual que Bokeh y mpld3. A da de hoy est principalmente enfocada a la generacin de grficos de series temporales. Se espera que consiga mucha mayor funcionalidad en el futuro.
