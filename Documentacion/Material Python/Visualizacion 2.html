1. Gráficos web sin D3: Bokeh¶

Bokeh es otra biblioteca de visualización basada en web que nos permite hacer gráficos bonitos e interactivos con poco esfuerzo, a partir de arrays de Numpy o Series y DataFrames de pandas.

A diferencia de D3, Bokeh no utiliza elementos SVG para dibujar en el navegador: sino que los genera en <canvas> de HTML5, lo cual los hace mucho más ligeros, y más apropiados si tenemos que representar una gran cantidad de datos. Por otro lado, el gráfico se genera más de la parte del servidor que la del cliente.

Bokeh se puede utilizar tanto en Notebooks de Jupyter, como en un servidor dedicado que sirva las visualizaciones a cualquier página web. Aquí nos vamos a centar en utilizarlo con los Notebooks.

Para instalar Bokeh, como siempre, podemos utilizar pip:

$ pip install bokeh

Una vez hecho: la primera vez que queremos que aparezca Bokeh en el Notebook, debemos inicializarlo:

In [2]:

Ahora bien, utilizar Bokeh es algo distinto a utilizar Matplotlib o Seaborn, pero tampoco demasiado. Al igual que en estos dos, debemos empezar por definir la figure en Bokeh (que en Matplotlib y Seaborn es opcional y lo hará por nosotros si no lo hacemos, pero en Bokeh solo en algunos casos):

In [3]:
from bokeh.plotting import figure as bokeh_figure, show # Le ponemos un alias a la
                                                        # importación de figure de
                                                        # Bokeh, para no confundir
                                                        # con el elemento figure
                                                        # de Matplotlib.

# Generamos un par de arrays de Numpy:
datos_x = np.arange(0, 50)
datos_y = np.random.randint(0, 100, 50)

# Hacemos una nueva figure de bokeh:
figura_bokeh = bokeh_figure(title="Ejemplo de Bokeh", x_axis_label='Eje x', y_axis_label='Eje y')

# Generamos una línea con los datos generados,
# y le damos un ancho de 3:
figura_bokeh.line(x=datos_x, y=datos_y, legend=u"Línea 1", line_width=3)

# Mostramos la figure con show:
show(figura_bokeh)
pass

Al igual que mpld3, Bokeh nos ofrece pequeños widgets para hacer zoom y movernos por el gráfico, y nos permite cambiar el tamaño del gráfico con el de "resize" una vez generado.

A pesar de que tiene poco más de año y medio, Bokeh es ya una biblioteca de visualización MUY extensa y de relativamente bajo nivel (quizás no tanto como Matplotlib, pero cada día se acerca más). Además, evoluciona increíblemente deprisa; principalmente porque Continuum Analytics (la empresa que distribuye Anaconda) está siendo su principal promotora.

Si bien se podría dedicar casi un módulo entero a Bokeh y vamos a dejar muchas de sus funcionalidades como trabajo "de campo" para el alumno, vamos a explicar a alto nivel los elementos principales de Bokeh.

Aparte del propio elemento figure (que al igual que en Matplotlib es como "el marco" de nuestros gráficos), Bokeh define principalmente tres elementos:

  • Charts: los elementos y visualizaciones estadísticos de alto nivel: barplots, gráficos de tarta, boxplots, mapas de calor, histogramas...
  • Glyphs: son elementos más a bajo nivel: círculos, líneas como la del primer ejemplo mostrado y similares. Dentro de la API se encuentran como métodos sobre la clase Figure, parte de bokeh.plotting: de forma que como en el primer ejemplo, primero definimos la figura, y luego llamamos a métodos de ella (como figura_bokeh.line()) para generarlos. Personalmente me parece una forma terrible de estructurar la biblioteca, pero para gustos los colores...
  • Widgets y otras interacciones: Bokeh a día de hoy es más una aplicación web que únicamente una biblioteca de visualización. Su extensa API construida sobre su propia biblioteca de Javascript permite, desde Python, generar una compleja lógica de interacción con los gráficos a través de la creación de widgets (como lupas de zoom, sliders, botones...), linkeado de varios gráficos (para que cambien a la vez cuando interactuamos con uno de ellos), incluso la generación de callbacks de Javascript (por ejemplo con AJAX) para dotar a tus visualizaciones de prácticamente todo lo que ofrece HTML5 en los navegadores modernos.

from bokeh.charts import Bar

# Y una paleta de color:
# http://bokeh.pydata.org/en/latest/docs/reference/palettes.html#bokeh-palettes
from bokeh.palettes import BrBG9

# En el fondo, los Charts de Bokeh hacen operaciones "por debajo",
# como son los groupby y value_counts:
barplot = Bar(data=df_padre[df_padre["tiempo trabajando"] >= 0], # Filtramos valores con pandas
              values="tiempo trabajando", # Nos centramos en la columna tiempo trabajando
              label="tiempo trabajando", # Hacemos un groupby por tiempo trabajando también (eje x)
              agg="count", # Queremos la cuenta
              stack="padre", # Queremos que en cada barra, nos stackee padre/no padre
              legend="top_right",
              color=BrBG9, # La palera de color importada
              title=u"Tiempo trabajado (frecuencia): Padre/No Padre",
              xlabel="Tiempo trabajando",
              ylabel="Cuenta",
              plot_width=700 # En píxeles
             )

show(barplot)
pass

# Cambiemos de paleta de color:
from bokeh.palettes import RdBu4

# Este RdBu4 es una lista de colores
# en hexadecimal que tiene 4 elementos,
# así que vamos a quedarnos con el primero
# y el último:

mi_paleta = [RdBu4[0],RdBu4[3]]

scatterplot = Scatter(data=df_padre,
                      x="renta",
                      y="edad",
                      color="padre",
                      marker="padre", # Hace que cada categoría tenga
                                      # una forma distinta
                      palette=mi_paleta,
                      legend="top_left",
                      title=u"Paternalidad",
                      xlabel="Renta",
                      ylabel="Edad",
                      background_fill_color="white", # Color de fondo
                      background_fill_alpha=0.4, # No negro al 100%
                      border_fill_color="yellow", # Color del marco
                      border_fill_alpha=0.3,
                      )

# Debido a una actualización reciente en Bokeh,
# para cambiar el tamaño del título y la fuente
# tenemos que hacerlo así, en vez de ponerlo 
# con el resto de atributos:
scatterplot.title.text_color="black"
scatterplot.title.text_font_size="24pt"

show(scatterplot)
pass

2.A. Widgets en Jupyter¶

Resulta que gracias al poder de Jupyter, también podemos hacer widgets de slider que interactúen con Matplotlib (por ejemplo). Gracias a la biblioteca ipywidgets podemos generar este tipo de interacciones. Podemos descargarla, como siempre, con pip:

$ pip install ipywidgets

Aquí está el Github del proyecto.

La mayoría de lo que podemos hacer lo obtenemos de ipywidgets.interactive:

In [11]:
from ipywidgets import interactive

Únicamente tenemos que incluir el contenido interactivo en una función, que tome como argumentos aquellos elementos que van a ser interactivos. Una vez definida nuestra función no tenemos más que llamar a interactive(nuestra_funcion, los_argumentos_que_puede_aceptar).

Podemos ver que en función de cómo pasemos los posibles argumentos (es decir, las posibilidades de cada variable que genera un widget) cada widget se adapta en forma para ofrecer la mejor interacción posible (slider, checkbox, selección)...

Nota: es posible que los widgets no aparezcan en la versión exportada (HTML) del notebook; y aún así, no serán funcionales. Para poder utilizar los widgets, es necesario abrir el notebook (el archivo .ipynb) y ejecutarlo.

In [12]:
def dibujar_histograma(numero_de_bins=5, color="blue", horizontal=False):
    # Configuramos el valor orientacion que le vamos a 
    # pasar luego a pandas para el plot:
    orientacion = "vertical"
    if horizontal == True:
        orientacion = "horizontal"

    # Y dibujamos en pandas (dentro de la misma
    # funcion):
    df_padre["edad"].plot(kind="hist",
                          bins=numero_de_bins,
                          color=color,
                          figsize=(8,6),
                          orientation=orientacion  # Los histogramas de pandas
                                                   # admiten el valor orientation
                                                   # que puede ser "horizontal"
                                                   # o "vertical" (por defecto). 
                        )

interactive(dibujar_histograma, numero_de_bins=(1, 100), color=["blue","red","green","yellow"], horizontal=False)

ipywidgets se puede utilizar allá donde tengamos una función; no tiene que ser necesariamente para hacer visualizaciones. Tiene un poder espectacular, y se espera que mejore más aún en el futuro.

2.B. Missingno¶

missingno es una biblioteca muy reciente que soluciona uno de los problemas más comunes en la ciencia de datos: identificar los valores que faltan (los missing, NaNs, o como los queramos llamar). Está evolucionando muy rápido, y recientemente (3 horas antes de que yo escribiera esto) han mejorado su funcionalidad.

Para instalar:

$ pip install missingno

Vamos a probarla. Vamos a generar una versión de nuestro df_padre con algunos NaNs:

In [13]:
def generar_nan(celda):
    """Va por cada celda de un dataframe,
    con la posibilidad del 30% de que dicha
    celda se convierta en NaN"""
    numero_random = np.random.randint(0,10)
    if numero_random <= 3:
        return np.nan
    else:
        return celda

# La aplicamos con applymap():
df_padre_con_nan = df_padre.applymap(generar_nan)

2.D. Otras bibliotecas de visualización¶

Cada vez aparecen más y más bibliotecas de visualización en Python. La mayoría de las nuevas están parcial o totalmente orientadas a la visualización web. Basta con buscar en Internet para encontrar, literalmente, más opciones de las que un ser humano equilibrado puede llegar a utilizar a lo largo de su vida.

Dicho esto, recomiendo echar un vistazo a las siguientes tres:

  • Lightning: De momento es muy joven, pero apunta a ser de las mejores para visualizaciones en tiempo real de forma sencilla; y compatibles con muchos lenguajes.
  • Pygal: Escasa en funcionalidad, pero es probablemente la más bonita de cuantas he visto. Incluye gráficos muy curiosos.
  • Plotly: Es bonita y sencilla, y bastante famosa. Ofrece herramientas para gente que no sabe programar; pero también ofrece una API de Python para los que sí saben (¡como tú!). Puedes comenzar leyendo sobre Plotly aquí
  • Bqplot: Desarrollada por Bloomberg, es muy reciente (salió hace tres meses aproximadamente). Permite generar visualizaciones interactivas en los Notebooks, al igual que Bokeh y mpld3. A día de hoy está principalmente enfocada a la generación de gráficos de series temporales. Se espera que consiga mucha mayor funcionalidad en el futuro.
